How it works

From idea to running task in seconds

No YAML hell. No infrastructure headaches. Just describe what you need.

1

Describe in plain English

Tell dicode what you want to automate. Use the CLI, the web UI, or even an AI agent via MCP. Natural language is all you need.

2

AI writes & commits the code

dicode generates a validated task script and commits it directly to your git repository — your single source of truth.

3

GitOps reconciliation

dicode watches your repo and auto-registers tasks from the tasks/ directory. Push a change — it's live within 30 seconds. Zero restarts.

4

Monitor & iterate

Watch live logs, inspect run history, trigger tasks manually, and chain them together — all from the web dashboard or REST API.

Write once, run everywhere with native runtimes

Every task gets SDK globals injected automatically — params, kv, output, dicode, mcp. Use native fetch() for HTTP, Deno.env.get() for secrets. TypeScript, Python, Docker, or Podman — pick the runtime that fits.

Or go full Docker: run any language in any container with live log streaming, kill support, and automatic orphan cleanup.

params kv store output dicode API fetch() secrets task chaining mcp
tasks/github-digest.ts 
// @name    GitHub Digest
// @cron    0 9 * * 1-5
// @desc    Post open PRs to Slack every weekday morning

const token = Deno.env.get("GITHUB_TOKEN");
const repo = await params.get("repo") ?? "my-org/my-repo";

const resp = await fetch(
  `https://api.github.com/repos/${repo}/pulls`,
  { headers: { Authorization: `Bearer ${token}` } }
);

const prs = await resp.json();
const last = await kv.get("last_pr_count") ?? 0;

if (prs.length !== last) {
  await fetch(Deno.env.get("SLACK_WEBHOOK"), {
    method: "POST",
    body: JSON.stringify({
      text: `📋 ${prs.length} open PRs in ${repo}`
    })
  });
  await kv.set("last_pr_count", prs.length);
}

console.log("digest sent", { count: prs.length });

Task SDK

Built-in globals for every runtime

Deno and Python tasks get SDK globals injected automatically. No imports needed.

params
Access typed task parameters. await params.get('key') in Deno, params.get('key') in Python
kv
Per-task persistent key-value store — survives restarts, tracks state across runs
input
Payload from upstream task (chain trigger) or incoming webhook request body
output
Render rich output — HTML, text, images, files — stored alongside the run
dicode
Run other tasks, list tasks, fetch run history, read config, manage secrets
mcp
Call tools on MCP daemon tasks — list_tools, call
env
Deno.env.get() or env.get() — only vars declared in task.yaml are accessible
